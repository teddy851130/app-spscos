# 통합 웹사이트 프로젝트 핸드오프

> 이 문서는 `app-spscos` (현재 내부 팀 전용 바이어 발굴 플랫폼)의 누적 자산을
> **새 통합 웹사이트 프로젝트**에서 이어받아 활용하기 위한 레퍼런스.
> 작성일: 2026-04-19. 작성 배경: PR13 클릭 추적 hotfix 과정에서 "URL 노출 이슈"가 드러나며 통합 사이트 분리 결정.

---

## 1. 통합 웹사이트가 갖춰야 할 역할

현재 두 가지가 분리되어 있음:
- `spscos.com` — 기존 하드코딩 마케팅 사이트 (정적)
- `app.spscos.com` — **이 프로젝트**. 내부 팀 전용 바이어 발굴·메일 발송 플랫폼

통합 웹사이트는 **`spscos.com`을 대체**할 외부용 공식 사이트. 핵심 기능:
1. 외부 바이어가 보는 **공식 마케팅 페이지** — 회사 소개·포트폴리오·연락처
2. **콜드 메일 P.S. 링크의 착지점** — 바이어가 링크 클릭 시 의심 없이 spscos.com 브랜드 페이지에 도착해야 함
3. **클릭 추적 redirect 엔드포인트** `/go/{token}` — 현재 `app.spscos.com/go/...`에 있는 기능을 통합 사이트 도메인으로 이전

내부 팀 플랫폼(`app.spscos.com`)은 **분리 유지**. 바이어에게 노출 금지.

---

## 2. 이어받을 핵심 기능 — 클릭 추적 시스템 (PR13 / ADR-032)

새 사이트에서 동일 기능을 그대로 이식 가능. 이미 안정화된 구조.

### DB 스키마 (Supabase, migration 011 완료)
- `buyer_contacts.tracking_token` TEXT UNIQUE — 12자 hex, DB 기본값 `encode(gen_random_bytes(6), 'hex')`
- `click_events` 테이블 — id, buyer_contact_id FK, clicked_at, user_agent, ip_address, ip_country, referer, pipedrive_status, pipedrive_activity_id, pipedrive_error
- RLS: click_events.SELECT true (대시보드 조회용), 쓰기는 service_role bypass

### Route Handler — `/go/[token]/route.ts` (재사용 가능)
파일: `app/go/[token]/route.ts`. Next.js 14 App Router, Node.js runtime, 약 100줄.

동작:
1. 토큰 정규식 검증 (`^[0-9a-f]{12}$`) — 위조 URL 빠른 폐기
2. buyer_contacts 조회 → click_events INSERT
3. contact_status 갱신 ('Interested'로. 단 Sample/Deal/Lost/Bounced 보호)
4. Pipedrive `/persons/search` → `/activities` POST (타임아웃 4초, 실패해도 redirect 계속)
5. 302 redirect → `https://<통합사이트>/` (현재는 `https://spscos.com/`)

통합 사이트에 그대로 이식하고, **fallback redirect 목적지만 바꾸면 됨** (또는 query param으로 지정).

### 재사용 가능한 라이브러리
- `app/lib/supabaseAdmin.ts` — service_role 지연 초기화, RLS bypass용
- `app/lib/pipedrive.ts` — Person 검색 + Activity POST + 타임아웃/에러 핸들링

### 필요한 환경변수
- `SUPABASE_SERVICE_ROLE_KEY` — RLS bypass (click_events INSERT + contact_status UPDATE)
- `PIPEDRIVE_API_TOKEN` — CRM Activity 생성
- `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` — 클라이언트 조회용

## 3. URL 구조 결정 (통합 사이트에서 재결정 필요)

### 현재 (임시)
`https://app.spscos.com/go/{12자 hex}` — 내부 도메인이라 바이어 노출 부적절

### 통합 사이트 런칭 시 후보 (런칭 시 Teddy 확정)
- **A.** `https://spscos.com/go/{token}` — 가장 자연스러움. 루트 도메인에서 `/go/` 경로만 확보.
- **B.** `https://go.spscos.com/{token}` — 전용 서브도메인. 링크 트래커처럼 보임.

통합 사이트가 Vercel에서 `spscos.com` 루트로 배포되면 **A안이 비용 최저**.

### 전환 시 실행 단계 (통합 사이트 런칭 시)
1. Supabase Edge Function Secret `TRACK_BASE_URL` 업데이트 (Dashboard → Functions → Secrets)
2. `run-pipeline/index.ts:613`, `generate-draft/index.ts:94` 하드코딩 fallback 정정 후 재배포
3. `validate-draft/index.ts:40-42` + `MailQueue.tsx:76` 스팸 검증 regex에 새 도메인 OR 추가 (legacy `app.spscos.com/go`와 `app-spscos.vercel.app/go`도 유지 → 과거 발송 메일 호환)
4. 기존 `app.spscos.com/go/{token}` 엔드포인트는 **절대 삭제하지 말고 유지** (이미 발송된 메일 클릭 수집)

## 4. 기술 스택 결정 (앞선 프로젝트에서 얻은 교훈)

### 확정된 선택
- **Next.js 14 App Router** (15 아님) — 현 `app.spscos.com`도 14.2.29. 런타임/params 시그니처 차이 주의
- **Vercel 배포** (GitHub main push 시 자동)
- **Supabase** (DB + Auth + Edge Functions Deno)
- **Tailwind CSS 4**
- **nodemailer@6.9.16** (denomailer 금지 — ADR 반복 언급)

### 피해야 할 것
- Edge Function에서 `denomailer` 사용 (ADR-009 교훈)
- 포괄 RLS 정책 DROP 후 SELECT/INSERT/UPDATE 개별 재생성 누락 (ADR-008 교훈)
- `supabase.functions.invoke` — direct fetch 사용 (ADR-009)
- **기본 Vercel 도메인(`*.vercel.app`) 하드코딩** — 커스텀 도메인 전환 후 응답 불가 상태가 됨 (PR13 hotfix 교훈)
- `buyers.status`에 한국어 직접 저장 — DB는 영어 enum, UI만 한국어 (`mapStatus` 경유)

## 5. 통합 시 고려사항

### 내부 플랫폼(`app.spscos.com`)과의 관계
- **공존 유지**: 통합 사이트(`spscos.com`)와 내부 플랫폼(`app.spscos.com`)은 분리 운영.
- **DB는 동일 Supabase 프로젝트 공유** — buyer_contacts, click_events, buyers 등 모두 공통. 통합 사이트에서는 **service_role**로 click_events 쓰기, 내부 플랫폼에서는 anon+authenticated로 조회.
- **인증 구조**: 내부 플랫폼은 `@spscos.com` Google Workspace 계정만. 통합 사이트는 공개 (인증 없음).

### 도메인·DNS
- 가비아에서 관리 (`spscos.com`)
- MX: `smtp.google.com` (Google Workspace)
- SPF/DKIM/DMARC 이미 설정됨 (2026-04-19 정비)
- **가비아 DNS 제약**: CNAME 값에 언더스코어(`_`) 포함 거부 → Managed 서비스 위임 불가, TXT 직접 등록 필수

### 메일 발송 연동
- 통합 사이트 런칭 이전에는 **실전 바이어 발송 보류 권장** — tracking URL이 `app.spscos.com` 노출되어 브랜드 리스크.
- 런칭 후 URL 전환 완료 시점부터 실전 발송 재개.
